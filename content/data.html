Your first steps in this project will be to find data to work on.

I recommend trying to find data that interests you and that you are knowledgeable about. A bad example would be if you have no interest in video games but your data set is about video games. I also recommend finding data that is related to current events, social justice, and other areas that have an impact.

You are not allowed to work on COVID data.

Initially, you will study one dataset but later you will need to combine that data with another dataset. For this reason, I recommend finding data that has some date and/or location components. These types of data are conducive to interesting visualizations and analysis and you can also combine this data with other data that also has a date or location variable. Data from the census, weather data, economic data, are all relatively easy to combine with other data with time/location components.

Where to keep data?

Below 50mb: In dataset folder

Above 50mb: In dataset_ignore folder. This folder will be ignored by git so you’ll have to manually sync these files across your team.

Sharing your data

For small datasets (<50mb), you can use the dataset folder that is tracked by github. Add the files just like you would any other file.

Do not use a folder named data. This folder is reserved by hugo. If you create a folder named data this will cause problems.

For larger datasets, you’ll need to create a new folder in the project root directory named dataset-ignore. This will be ignored by git (based off the .gitignore file in the project root directory) which will help you avoid issues with Github’s size limits. Your team will have to manually make sure the data files in dataset-ignore are synced across team members.

The main dataset used in our analysis of factors that influence a win among Major League Baseball teams is the Baseball Game Logs from 1871-2016 which is a dataset that can be found on the Data.World website. This dataset was compiled from 127 separate CSV files from Retrosheet, a website dedicated to providing as many play-by-play accounts of major league games as possible. Many people have contributed to providing the game logs, however, baseball historian Bob Tiemann is credited with originating the logs and Arnie Braunstein is credited with computerizing them. The data from Retrosheet was collected for the purpose of providing fans access to data that allows them to compare player and team performances to other years.

Cleaning

A link to our load_and_clean_data.R file can be found here.

This dataset is >50mb and can therefore be found in our dataset-ignore folder. After reading in the CSV file, we specified the column types for variables forfeit and completion to remove any problems. We converted the dataset into a tibble and saved it under the variable baseball.

Our next step in cleaning the data was to take the date as an integer and separate it into three columns: year, month, and day.

Due to the largeness of the dataset as well as significant missing data from the 1970s and 1990s, we chose to work with more recent years and so filtered the data to only include those baseball games that took place between the years 2000 and 2016.

The last step in cleaning our baseball dataset was to mutate a column titled score_diff. v_score and h_score provides the number of runs scored by each team, but the dataset does not include a variable that states which team won a given game. score_diff takes the number of the visiting team’s runs subtracted from the number of the home team’s runs. A positive value represents a win for the home team, a negative value represents a loss, and a zero is a tie. The higher absolute value of score_diff represents a larger win or loss.

In order to analyze the effects of teams’ payrolls on their success each season, we compiled a csv file based on data found from the “MLB Team Payrolls” link found in our “DATASETS” table. The website itself did not contain any easily downloadable data file, so we copied the columns “Team”, “Rank”, and “Total (Payroll)” from the years 2000-2016 into a csv file, adding a column indicating the year for each observation. It is important to note that not each year has consistent column names on the website, so we consistently interpreted the highest value payroll columns as the “Total” payroll values. Additionally, since not each year had a “Rank” column, we manually observed and added “Rank” values for those that were missing. Lastly, we adjusted the team names to match our main dataset’s unique h_name abbreviations. This csv file can be found in our dataset folder.
